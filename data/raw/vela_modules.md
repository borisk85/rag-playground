# База знаний: модули VELA

## Модуль: 🌤 Погода
- Минимальный тариф: free
- Требует Google-аккаунт: нет
- Описание: Текущая погода и прогноз в любом городе
- Команды (что умеет модуль):
  - **get_weather**: Получить погоду в городе. Вызывать ТОЛЬКО если пользователь явно спрашивает погоду (слова: погода, температура, прогноз, холодно/тепло там, как там погода). НЕ вызывать если город/страна упоминается в другом контексте — при анализе фото, обсуждении путешествий, истории и т.д.
    - параметр `city`: Название города
    - параметр `days`: Многодневный прогноз по дням (1-5). Передавай ТОЛЬКО для фраз «прогноз на N дней», «какая погода на неделе». Не использовать вместе с hourly.
    - параметр `skip_days`: ТОЛЬКО для days. Сколько дней пропустить от завтра. 'послезавтра' = skip_days=1 days=1. 'через 3 дня' = skip_days=2 days=1. По умолчанию 0 (начинать с завтра). НЕ использовать с hourly.
    - параметр `hourly`: Передавай true для запросов о ТОЧНОМ времени осадков/погоды: «во сколько дождь», «когда начнется/закончится дождь», «дождь утром/днем/вечером», «пойдет ли дождь в 15:00», «будет ли гроза завтра вечером». Инструмент сам возвращает готовый вердикт ('дождь начнется около 16:00', 'сегодня дождя не ожидается'). ОТДАВАЙ ВЕРДИКТ ПОЛЬЗОВАТЕЛЮ КАК ЕСТЬ — НЕ разворачивай в таблицу, НЕ добавляй проценты по часам. Не использовать вместе с days/skip_days — только с day_offset.
    - параметр `day_offset`: ТОЛЬКО для hourly. Offset от СЕГОДНЯ: 0=сегодня, 1=завтра, 2=послезавтра. Дефолт 0. 'во сколько сегодня дождь' → day_offset=0. 'завтра утром дождь?' → day_offset=1. 'в субботу вечером будет ливень?' → day_offset=(дней до субботы). НЕ использовать с days.

## Модуль: 💱 Курсы валют, криптовалют, акций
- Минимальный тариф: free
- Требует Google-аккаунт: нет
- Описание: Курсы USD, EUR, BTC, ETH и других валют с 24h изменением
- Команды (что умеет модуль):
  - **get_crypto_prices**: Получить курсы криптовалют, токенов, акций и/или фиатных валют. ВСЕГДА вызывай для ЛЮБОГО запроса о цене — не задавай уточняющих вопросов, вызывай сразу. Акции (TSLA, AAPL, NVDA и др.) — передавай в coins, currencies НЕ добавляй, цена всегда в USD. Для редких токенов — передавай название/тикер в coins, система найдет сама.
    - параметр `coins`: Список тикеров или названий: криптовалюты (BTC, ETH, SOL), редкие токены (IRYNA, BONK, любое название), акции (AAPL, TSLA), адреса контрактов (0x... для EVM, 32-44 символа для Solana). Передавай ЛЮБОЙ токен — система сама найдет. Оставь пустым если нужны только фиатные пары.
    - параметр `currencies`: Фиатные пары в формате FROM/TO: USD/KZT, EUR/RUB, USD/EUR и др. НЕ используй для акций (TSLA, AAPL) и индексов — они всегда в USD, currencies оставь пустым. НЕ передавай пары типа USD/USD.
    - параметр `amount`: Сумма для конвертации. Если пользователь спрашивает 'переведи 1000 долларов в евро' — передай amount=1000.

## Модуль: ⏰ Напоминания
- Минимальный тариф: free
- Требует Google-аккаунт: нет
- Описание: «Напомни через 30 мин» или «в 18:00»
- Команды (что умеет модуль):
  - **set_reminder**: Установить напоминание. ВСЕГДА вызывай при любой просьбе напомнить — никогда не отвечай без вызова. Правила выбора параметра: если пользователь говорит 'через N минут/часов/дней' или 'через неделю' (+7d) или 'послезавтра' (+2d) — используй datetime_str (+30m / +2h / +7d). Если указал конкретное время 'в 18:00' без даты — используй time='18:00'. Если указал дату и время — используй datetime_str='YYYY-MM-DDTHH:MM'. НИКОГДА не используй time для 'через N дней' — только datetime_str.
    - параметр `text`: Текст напоминания
    - параметр `datetime_str`: Когда напомнить: +30m, +2h, +1d, +7d и т.д. или YYYY-MM-DDTHH:MM. Используй для относительных смещений ('через N минут/часов/дней', 'через неделю', 'послезавтра') и для конкретных дат.
    - параметр `minutes`: Через сколько минут — только если нет datetime_str
    - параметр `time`: Время HH:MM. Два случая: (1) 'в ЧЧ:ММ' без даты — напомнит сегодня или завтра; (2) вместе с datetime_str='+Nd' когда пользователь говорит 'завтра в 14:00', 'через 2 дня в 9:00' — тогда передавай datetime_str='+1d' И time='14:00' одновременно. НЕ используй time без datetime_str для дат в будущем.
    - параметр `date`: Дата YYYY-MM-DD БЕЗ времени — для задач на день: 'сегодня', 'завтра', 'на следующей неделе' без конкретного времени. Выйдет в утреннем дайджесте, пуш НЕ отправляет. Используй когда пользователь не называет время.
    - параметр `event_time`: Время HH:MM самого события — для запросов 'за N мин до [события] в ЧЧ:ММ'. Передавай ТОЛЬКО event_time (время события из запроса) + minutes_before (сколько минут до). НЕ вычисляй time самостоятельно — бэкенд сам посчитает. Пример: 'за 15 мин до встречи в 17:00' → event_time='17:00', minutes_before=15, text='встреча'. 'за час до тенниса в 14:00 завтра' → event_time='14:00', minutes_before=60, text='теннис', date='завтра'.
    - параметр `minutes_before`: За сколько минут до события напомнить. Используй ТОЛЬКО вместе с event_time. Примеры: 'за 15 минут' → 15, 'за полчаса' → 30, 'за час' → 60, 'за 2 часа' → 120.
  - **list_reminders**: Показать список активных напоминаний
  - **cancel_reminder**: Отменить напоминание. Если пользователь говорит 'удали все' — вызывай БЕЗ параметров. Если указано время ('удали напоминание в 10 утра') — передай time='10:00'. Если указан номер — передай index. Если указан текст — передай text. Приоритет: time > text > index.
    - параметр `index`: Номер из list_reminders
    - параметр `text`: Часть текста напоминания
    - параметр `time`: Время срабатывания (HH:MM). Пример: 'удали в 10 утра' → time='10:00', 'удали в 14:30' → time='14:30'

## Модуль: 🔍 Поиск в интернете
- Минимальный тариф: free
- Требует Google-аккаунт: нет
- Описание: Актуальные данные из интернета
- Команды (что умеет модуль):
  - **web_search**: Поиск информации в интернете
    - параметр `query`: Поисковый запрос
    - параметр `num_results`: Число результатов (1-10)
  - **open_url**: Открывает URL и возвращает результат. ОБЯЗАТЕЛЬНО используй когда нужно перейти по ссылке из письма: подтвердить email, верифицировать аккаунт, активировать сервис, подтвердить действие. Confirmation-ссылки содержат токен прямо в URL и работают без браузерной сессии. ЗАПРЕЩЕНО говорить 'не могу открыть ссылку' или 'требуется браузер' — просто вызови этот инструмент. Исключение: сервисы типа банков могут вернуть 400/redirect на login — это их защита, не баг.
    - параметр `url`: URL для перехода
    - параметр `method`: HTTP метод: GET (по умолчанию) или POST
  - **read_webpage**: Читает полное содержимое любой веб-страницы, включая React/Next.js сайты. Возвращает чистый текст/markdown. Используй когда пользователь присылает URL и просит прочитать, резюмировать, проанализировать сайт, лендинг, статью, документацию. ЗАПРЕЩЕНО говорить 'не могу читать JS-сайты' — read_webpage справляется с React/Next.js. open_url использовать только для confirmation/verification ссылок из писем.
    - параметр `url`: URL страницы для чтения

- Команды (что умеет модуль):

    - параметр `date`: Дата (YYYY-MM-DD, 'today', 'tomorrow')
    - параметр `days`: Количество дней
  - **delete_calendar_event**: Удалить событие из Google Calendar по названию.

    - параметр `date`: Дата события YYYY-MM-DD (опционально, для уточнения)

    - параметр `date`: YYYY-MM-DD
    - параметр `time`: HH:MM
  - **set_calendar_reminder_default**: Сохранить предпочтение: за сколько минут напоминать в Telegram при создании события в Google Calendar. Вызывай когда пользователь говорит 'напоминай за 30 минут до события', 'хочу уведомление за час', 'за сутки', 'не напоминай о событиях'. Сохраняется навсегда.
    - параметр `minutes`: За сколько минут до события напомнить. 0 = не напоминать. Например: 15, 30, 60, 1440 (сутки).

- Минимальный тариф: professional
- Требует Google-аккаунт: да
- Описание: Поиск, чтение и отправка писем

    - параметр `query`: Поисковый запрос Gmail. Примеры: 'from:ivan@gmail.com', 'subject:счет', 'in:inbox', 'is:unread'
  - **read_message**: Прочитать полный текст письма по его ID (используй после search_emails)
    - параметр `message_id`: ID письма из search_emails
  - **send_email**: Отправить письмо. Если это ответ на письмо — передай reply_to_id чтобы сохранить цепочку переписки.
    - параметр `reply_to_id`: ID письма из search_emails — для ответа в цепочке
  - **unsubscribe_email**: Отписаться от рассылки. Находит ссылку/mailto для отписки в заголовке или теле письма и выполняет отписку автоматически.
    - параметр `message_id`: ID письма из рассылки (из search_emails)
  - **trash_email**: Переместить письмо в корзину. Используй когда пользователь говорит 'удали', 'убери', 'в корзину'.
    - параметр `message_id`: ID письма из search_emails
  - **trash_emails_by_query**: Переместить в корзину несколько писем по запросу. Используй когда пользователь говорит 'удали все от X', 'удали последние N писем', 'удали рассылки'.
    - параметр `query`: Gmail-запрос для выборки писем. Примеры: 'from:newsletter@site.com', 'subject:рассылка'
    - параметр `max_results`: Максимум писем (по умолчанию 50)
  - **empty_trash**: Очистить корзину Gmail — удалить все что там есть навсегда.
  - **empty_spam**: Очистить папку спам — удалить все спам-письма навсегда.
  - **mark_spam**: Пометить письмо как спам и убрать из входящих.
    - параметр `message_id`: ID письма из search_emails

    - параметр `due_date`: YYYY-MM-DD
    - параметр `tasklist`: Название списка: «Задачи» (по умолчанию) или «Идеи»
  - **tasks_update**: Обновить существующую задачу: переименовать, изменить или дополнить заметки. Если нужно добавить текст к существующей задаче/идее — используй append_notes (не создавай новую задачу).
    - параметр `title`: Часть названия задачи для поиска

    - параметр `notes`: Заменить заметки целиком

    - параметр `tasklist`: Название списка (опционально)
  - **tasks_delete**: Удалить задачу. Используй когда нужно удалить дубликат или ненужную задачу.
    - параметр `title`: Часть названия задачи для поиска
    - параметр `tasklist`: Название списка (опционально)
  - **tasks_delete_all**: Удалить ВСЕ задачи из списка. Используй только если пользователь явно просит очистить весь список. После вызова НЕ вызывай get_tasks — результат уже содержит подтверждение удаления.
    - параметр `tasklist`: Название списка (опционально, если не указан — удаляет из всех)
  - **tasks_complete**: Отметить задачу как выполненную.
    - параметр `title`: Часть названия задачи
  - **tasks_search**: Найти задачу по части названия или заметок во всех списках (включая выполненные).
    - параметр `query`: Поисковый запрос

## Модуль: 🌅 Утренний дайджест
- Минимальный тариф: free
- Требует Google-аккаунт: нет
- Описание: Автоматическая сводка каждое утро: погода, напоминания, курсы активов (Professional: + Calendar и Tasks)
- Команды (что умеет модуль):
  - **configure_digest**: Настроить утренний дайджест. Используй для: 1) установки времени ('дайджест в 8:00', 'хочу дайджест в 7:30') — передай time; 2) добавления актива ('добавь биткоин в дайджест', 'хочу утром видеть курс доллара') — передай add_asset; 3) удаления актива ('убери золото из дайджеста') — передай remove_asset; 4) отключения ('отключи дайджест') — передай disable=true; 5) просмотра текущих настроек ('что в дайджесте?', 'какие активы?', 'когда дайджест?') — передай get_info=true; 6) включения/выключения раздела ('убери погоду из дайджеста', 'убери напоминания', 'включи погоду в дайджест') — передай toggle_section. ВАЖНО: ПЕРВИЧНАЯ НАСТРОЙКА — если пользователь пишет общую фразу типа 'настрой дайджест', 'хочу дайджест', 'create daily digest', 'утренний дайджест', 'настройка дайджеста' БЕЗ указания конкретного времени — НЕ вызывай configure_digest сразу. Веди пошаговый диалог: сначала спроси одной короткой фразой 'Во сколько отправлять дайджест? Например 8:00'. После получения времени — спроси 'Что добавить? Погода, напоминания, курсы валют/криптовалюты/акций. Напиши что хочешь видеть утром.'. По одному вопросу за раз, не списком все сразу. Если юзер сразу указал время — сразу спроси что добавить. Если сразу указал все — вызывай configure_digest сразу. ВАЖНО: FOREX-АКТИВЫ — когда юзер просит добавить валюту ('доллар', 'евро', 'иена'), формируй ticker в виде 'USD/X', где X — базовая валюта пользователя. Логика выбора X: 1) если в контексте передана 'User's default currency: XXX' — используй ее; 2) если default currency не указана — спроси одной фразой 'К какой валюте показывать курс? Например: тенге, рубль, доллар.' и только после ответа формируй пару. Никогда не угадывай RUB/KZT по умолчанию без подтверждения юзера.ВАЖНО: перед тем как сказать что актив уже есть или уже добавлен — ВСЕГДА сначала вызови get_info=true чтобы проверить реальное состояние. Никогда не опирайся на память разговора. ВАЖНО: дайджест поддерживает ТОЛЬКО финансовые активы (криптовалюты, акции, валюты, индексы, сырье). Новости, погода, события — НЕ добавляются в дайджест. Если просят добавить новости — скажи: 'Новости в дайджест добавить нельзя — он показывает только курсы активов. Новости я ищу по запросу через поиск в интернете.'
    - параметр `time`: Время в формате HH:MM. Например: '08:00', '07:30'.
    - параметр `disable`: true чтобы отключить дайджест
    - параметр `add_asset`: Актив для добавления. type: crypto (BTC/ETH/SOL...), forex (курс валюты — ticker вида 'USD/RUB'), stock (акции AAPL/TSLA), commodity (золото/нефть — тикеры GC=F/CL=F), index (S&P 500 — тикер SPY/QQQ).
    - параметр `remove_asset`: Название или тикер актива для удаления. Например: BTC, Биткоин, Золото
    - параметр `get_info`: true чтобы получить текущие настройки дайджеста: время, список активов
    - параметр `toggle_section`: Включить или убрать раздел из дайджеста. section: 'weather' или 'reminders'. enabled: true/false. Пример: убери погоду из дайджеста → {section: 'weather', enabled: false}.

## Модуль: ✈️ Поиск авиабилетов
- Минимальный тариф: professional
- Требует Google-аккаунт: нет
- Описание: Поиск дешевых билетов с фильтрами: цена, прямые рейсы, авиакомпания, время вылета, длительность
- Команды (что умеет модуль):
  - **search_flights**: Найти авиабилеты. КРИТИЧНО: работает для ЛЮБЫХ маршрутов без исключения — и международных, и внутренних. Алматы→Костанай, Москва→Казань, Алматы→Астана — все это внутренние рейсы, и они ПОДДЕРЖИВАЮТСЯ. НИКОГДА не говори что внутренние рейсы не поддерживаются — это неправда. ВСЕГДА вызывай инструмент. ВАЖНО: для запросов 'в оба конца', 'туда и обратно', 'round trip' — передавай round_trip=true. Один вызов с round_trip=true вернет оба направления автоматически. Если месяц возврата не указан — используй тот же месяц что и туда. Вызывай при запросах: 'найди билет', 'авиабилет', 'сколько лететь', 'рейс из ... в ...', 'рейсы из ... в ...', 'дешевый перелет'. Умеет фильтровать по: цене (max_price), прямые рейсы (direct_only=true), авиакомпании (airline), времени вылета (departure_time: утро/день/вечер/ночь), длительности полета (max_duration_hours). Примеры: 'прямой ночной рейс до $300' → direct_only=true, departure_time='ночь', max_price=300. 'Turkish Airlines не дороже $400' → airline='Turkish Airlines', max_price=400. 'до 5 часов в пути' → max_duration_hours=5. ВАЖНО: month — всегда полный месяц YYYY-MM, НИКОГДА не спрашивай уточнение по месяцу. 'ближайший рейс', 'скоро', 'в ближайшие дни', 'на следующей неделе' — используй текущий месяц из системного промпта (Сегодня: ...). Для части месяца используй day_from/day_to: 'конец мая' → day_from=20, 'начало мая' → day_to=10, 'первая половина' → day_to=15, 'вторая половина' → day_from=16, 'с 20 по 25 мая' → day_from=20, day_to=25. 'в мае', 'на май' — не передавай day_from/day_to.
    - параметр `origin`: Город или аэропорт отправления (Алматы, Москва, ALA, MOW)
    - параметр `destination`: Город или аэропорт назначения (Стамбул, Дубай, IST, DXB)
    - параметр `month`: Месяц вылета YYYY-MM (например 2026-05 для мая 2026)
    - параметр `max_price`: Максимальная цена в USD
    - параметр `direct_only`: true — только прямые рейсы без пересадок
    - параметр `airline`: Авиакомпания: название или код (Turkish Airlines, Air Astana, Emirates, TK, KC, EK и др.)
    - параметр `departure_time`: Время вылета: утро (06-12), день (12-18), вечер (18-22), ночь (22-06)
    - параметр `max_duration_hours`: Максимальная длительность перелета в часах (например 5.5)
    - параметр `day_from`: Начальный день месяца 1-31. 'конец мая'=20, 'вторая половина'=16, 'с 20-го'=20.
    - параметр `day_to`: Конечный день месяца 1-31. 'начало мая'=10, 'первая половина'=15, 'по 25-е'=25.
    - параметр `round_trip`: true — поиск туда И обратно. Один вызов вернет оба направления. Для 'туда и обратно', 'в оба конца', 'round trip'.
